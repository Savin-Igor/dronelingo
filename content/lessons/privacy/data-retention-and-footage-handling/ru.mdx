## Базовое правило: сначала цель, потом хранение

Если материал собран для определённой цели, он должен храниться только ради этой цели и только пока эта цель существует.

Для оператора дрона это означает разделять:

- **клиентские deliverables**;
- **сырой исходный материал**;
- **операционные доказательства**, которые нужны для безопасности, отчётности или страхования;
- **материал, который нужно удалить**, потому что у него больше нет продолжающейся цели.

Хранить всё навсегда — не нейтральное решение. Это увеличивает приватностный и security-риск.

## Сырой материал — не автоматический архив

Операторы часто относятся к SD-карте или экспортной папке как к постоянному архиву. Это плохая практика.

Лучший workflow такой:

1. импортировать материал в структурированную проектную папку;
2. отделить финальные deliverables от сырых файлов;
3. оставить только минимальный набор сырого материала, нужный для монтажа, договорного подтверждения или доказательств по инциденту;
4. удалять дубликаты и бесполезный материал по понятному графику.

## Расшаривание создаёт новый риск

Обращение с материалом становится опаснее, когда оператор:

- пересылает клипы через открытые мессенджеры;
- отдаёт клиенту весь дамп карты;
- автоматически синхронизирует сырой материал в неконтролируемые облачные аккаунты;
- хранит годы идентифицируемого материала на незашифрованных дисках.

Каждая лишняя копия — это ещё одна потенциальная утечка.

## Когда хранение оправдано

Более длительное хранение может быть оправдано, если материал нужен для:

- клиентского договора;
- спора;
- страхового требования;
- сообщения об инциденте;
- юридической обязанности.

Но причина “вдруг когда-нибудь пригодится” слабая.

<Scenario title="После съёмки стройплощадки у оператора осталось 90 минут сырого материала. Клиенту нужен только 90-секундный ролик. Несколько клипов показывают соседние сады и людей, не связанных с работой. Какой подход к обработке правильный?">
Материал нужно хранить только в соответствии с целью.

Оператору следует:

1. выделить материал, реально нужный для клиентского результата;
2. определить, нужно ли временно оставить часть сырого материала для монтажа или договорного подтверждения;
3. удалить нерелевантные клипы с лишним захватом третьих лиц, как только они перестанут быть нужны;
4. не отдавать клиенту весь сырой дамп карты, если на это нет ясного договорного основания.

Цель не в нулевом хранении. Цель — в обоснованном хранении.
</Scenario>

<CommonMistakes>
- Хранить каждый дамп SD-карты вечно “на всякий случай”.
- Отправлять полные папки сырого материала через потребительские мессенджеры.
- Считать, что весь отснятый материал автоматически входит в клиентский пакет.
- Забывать, что резервные копии тоже считаются хранимыми персональными данными.
- Держать старый материал на незашифрованных дисках без графика хранения.
</CommonMistakes>
